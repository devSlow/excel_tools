# 公告配置接口需求文档

## 一、背景说明

### 现有参考模块：轮播图（Banner）

| 项目 | 说明 |
|------|------|
| 接口地址 | `GET /banner` |
| 返回格式 | `{ code: 0, data: [{ id, imageUrl, linkUrl }, ...] }` |

### 新增需求

在文本解析页（TabBar 首页）新增公告展示功能，支持图文混排（富文本），由后台管理系统管理公告内容。

---

## 二、接口定义

### 获取公告列表

- **接口地址：** `GET /notice`
- **是否需要鉴权：** 是（Bearer Token）
- **返回格式：**

```json
{
  "code": 0,
  "data": [
    {
      "id": 1,
      "title": "系统维护通知",
      "content": "<p style=\"text-align:center\"><strong>重要通知</strong></p><p>2025年4月25日凌晨2:00-6:00进行系统维护，届时服务暂停。</p><p><img src=\"https://example.com/notice.png\" style=\"width:100%\" /></p>",
      "type": "info",
      "enabled": true,
      "createdAt": "2025-04-23T10:00:00Z",
      "updatedAt": "2025-04-23T10:00:00Z"
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | number | 是 | 公告ID |
| `title` | string | 是 | 公告标题，纯文本 |
| `content` | string | 是 | 公告内容，HTML 富文本，支持图文混排 |
| `type` | string | 否 | 公告类型：`info`（一般通知）/ `warning`（警告）/ `error`（紧急），前端用于区分展示样式 |
| `enabled` | boolean | 是 | 是否启用，前端只展示 `enabled=true` 的公告 |
| `createdAt` | string | 否 | 创建时间，ISO 8601 格式 |
| `updatedAt` | string | 否 | 更新时间，ISO 8601 格式 |

### 业务规则

- 前端只请求列表接口，公告的增删改由管理后台完成
- `enabled=false` 的公告前端不展示
- 支持多条公告，前端按时间倒序展示
- 接口风格与现有 `/banner` 保持一致，统一返回 `{ code: 0, data: [...] }`

## 三、content 字段富文本规范

前端使用微信小程序 `rich-text` 组件渲染 `content` 字段，该组件支持以下 HTML 子集。

### 支持的 HTML 标签

| 标签 | 用途 | 示例 |
|------|------|------|
| `<p>` | 段落 | `<p>正文内容</p>` |
| `<strong>` / `<b>` | 加粗 | `<strong>重点内容</strong>` |
| `<em>` / `<i>` | 斜体 | `<em>补充说明</em>` |
| `<u>` | 下划线 | `<u>强调文本</u>` |
| `<span>` | 行内样式 | `<span style="color:red">红色文字</span>` |
| `<img>` | 图片 | `<img src="url" style="width:100%" />` |
| `<br>` | 换行 | `第一行<br/>第二行` |
| `<h1>` ~ `<h6>` | 标题 | `<h3>小标题</h3>` |
| `<ul>` / `<li>` | 无序列表 | `<ul><li>项1</li><li>项2</li></ul>` |
| `<ol>` / `<li>` | 有序列表 | `<ol><li>步骤1</li><li>步骤2</li></ol>` |

### 支持的内联样式

| 样式 | 用途 | 可选值 / 示例 |
|------|------|---------------|
| `text-align` | 对齐方式 | `left` / `center` / `right` |
| `font-weight` | 字体粗细 | `bold` / `normal` |
| `color` | 文字颜色 | `#ff0000` / `red` |
| `font-size` | 字号 | `14px` / `16px` / `20px` |
| `background-color` | 背景色 | `#fff3cd` / `transparent` |
| `width` | 宽度（图片） | `100%` / `auto` |
| `line-height` | 行高 | `1.5` / `2` |

### 富文本示例

#### 示例一：居中加粗 + 图片

```html
<p style="text-align:center"><strong>系统升级通知</strong></p>
<p>本次升级新增数据导出功能，欢迎体验。</p>
<p><img src="https://cdn.example.com/upgrade.png" style="width:100%" /></p>
```

#### 示例二：列表 + 彩色标签

```html>
<p><strong>更新内容：</strong></p>
<ul>
  <li>新增文本解析功能</li>
  <li>优化 Excel 导出速度</li>
  <li><span style="color:red">修复已知问题</span></li>
</ul>
```

#### 示例三：左对齐段落 + 分步骤说明

```html>
<h3>操作指南</h3>
<ol>
  <li>打开小程序，进入解析页面</li>
  <li>粘贴或输入需要解析的文本</li>
  <li>点击"开始解析"按钮</li>
</ol>
```

## 四、管理后台建议

- 建议集成富文本编辑器（如 [TinyMCE](https://www.tiny.cloud/)、[wangEditor](https://www.wangeditor.com/)），方便运营人员可视化编辑
- 编辑器工具栏建议包含：**加粗、斜体、下划线、对齐方式、有序列表、无序列表、插入图片、文字颜色**
- 公告管理页面字段：标题（文本输入）、内容（富文本编辑器）、类型（下拉选择）、启用状态（开关）
- 提供预览功能，确保在小程序端展示效果正常

---

## 五、前端展示位置

- 公告展示位置：文本解析页（TabBar 首页），位于轮播图 Banner 下方
- 展示形式：使用 `swiper` 轮播展示公告卡片（展示公告标题），多条公告自动轮播切换
- 点击公告卡片后跳转到公告详情页（独立页面），详情页使用 `rich-text` 渲染富文本内容
- 交互方式与现有轮播图 Banner 一致：swiper 展示 → 点击 → 页面跳转
